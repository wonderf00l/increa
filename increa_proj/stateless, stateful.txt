stateful системы - сохраняют состояние на сервере
stateless - перекладывают это на клиентскую часть

stateful плохо масштабируемы

частям системы, которые при масштабировании поддаются репликации, придется обращаться к тем или иным сервисам, хранящим информацию о клиенте, для корректной обработки запросов ВСЕМИ репликантами

например, redis и тп

в контексте микросервисов такой подход малоприменим, в контексте монолитной системы - да


в stateless системах пользователь получает token при успешной аутентификации, этот токен известен всем серверам, он обрабаывается одним из серверов



аутентификация - получение прав доступа к ресурсу посредством предоставления логина и пароля
error status: 401 unauthorized

авторизация - получение прав конкретного уровня, процесс их выдачи и проверки - авторизация
error status: 403 forbidden
